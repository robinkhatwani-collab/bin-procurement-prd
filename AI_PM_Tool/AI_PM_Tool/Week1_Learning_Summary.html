Types of AI — The Progression

📈 Useful Progression to Remember

Predict → Create → Act  — Each generation of AI became more capable by doing more than the last.

01 · Predictive AI

🔮 Predict

Data In → Prediction Out

Looks backward — learns from historical data to make forecasts or classifications. Doesn't create anything new.

Examples: Credit scores, spam filters, rain chances
Structured Data In Score / Label Out
02 · Generative AI

✨ Create

Prompt In → Content Out

Creates something new from a prompt. Trained on massive datasets — produces text, images, code, or audio.

Examples: ChatGPT, Claude, GitHub Copilot, Sora (Video)
Natural Language In New Content Out
03 · Agentic AI

⚡ Act

Goal In → Actions Out

Takes autonomous action. Combines reasoning with tools (web search, APIs, code) to complete multi-step tasks.

Examples: AI research agents, auto-coding agents, AI workflows
High-Level Goal In Real-World Actions Out

Rule-Based vs ML Systems

Why this comes before the math

Before diving into equations, it helps to understand why machine learning exists at all. Rule-based systems were the original approach — but they hit hard limits. Understanding those limits explains why ML was invented.

Dimension 🔧 Rule-Based Systems 🤖 ML-Based Systems
Decision MethodPredefined rules & logic written by humansLearns patterns from data; improves over time
Data RequiredSmall, structured datasetsLarge quantities of unstructured data
UpdatesManual rule updates needed by engineersAuto-adapts to new data & changing conditions
HardwareStandard CPU — runs anywhereSpecialized GPU / TPU + high-performance computing
ComplexityLow — logic is explicit and auditableHigh — uses statistical models & algorithms
ExplainabilityFully explainable — logic is human-written"Black box" — consider second-level classifiers to improve explainability
BiasHuman bias baked into rulesCan be designed to detect and mitigate bias
InteroperabilityGenerally more interoperable across systemsCan be less interoperable; requires careful API design

🤔 PM Decision Guide: When to Use Which?

✅ Choose Rule-Based When:
  • Small, structured data available
  • No ML engineers in the team
  • Low infrastructure budget
  • Need full explainability & auditability
✅ Choose ML-Based When:
  • Large, unstructured data available
  • Anticipating significant growth in scale
  • ML expertise available in team
  • Can leverage pre-trained models to reduce cost
📌 Always include a Data Collection Strategy in your product plan

Regardless of which approach you choose, your product spec must define how data will be collected. For ML systems with black-box outputs, consider adding second-level classifiers to make results explainable to users and stakeholders.

The Equation Behind All of AI

y = Wx + b

This simple equation is the universal primitive of all modern AI.
Every neural network, at every layer, runs this equation — repeated billions of times.

W = Weight (how important) x = Input Data b = Bias (default lean) y = Output (prediction)

🔑 Key Insight — Why Deep Learning is Powerful

A single y = Wx + b can only draw a straight line and solve simple problems. The magic of deep learning comes from stacking many layers with non-linear activation functions (like ReLU) between each layer. That combination lets the network learn curves, spirals, and patterns of any shape — enough to recognise faces, understand language, or beat world champions at Go.

💡 GPT = Generative Pre-trained Transformer

Modern large language models like GPT are built on this same fundamental equation, scaled up to billions of parameters with transformer architecture and attention mechanisms layered on top.

📧 Real Example: Spam Filter using y = Wx + b

Each word in an email is given a weight (W) reflecting how strongly it signals spam. The bias (b) sets the default — assume innocent until proven guilty.

spam_score = W₁×"free" + W₂×"winner" + W₃×"unsubscribe" + W₄×"meeting" + b W("free") = +2.1 → strong spam signal 🚨 W("winner") = +1.9 → spam signal 🚨 W("unsubscribe") = 0.3 → weak signal W("meeting") = -1.8 → work email, not spam ✅ b = -0.5 → default: assume NOT spam spam_score = 2.1×1 + 1.9×1 + 0.3×0 + (-1.8×0) + (-0.5) spam_score = 3.5 → SPAM (threshold > 0.5) ✉️🚫

Machine Learning Models

🔑 Three Things to Build Any ML Product

Data (the fuel)  ·  Training (the learning process)  ·  Inference (applying the trained model to new inputs)

🧩 What Exactly is a "Model"?

A model is the equation y = Wx + b with trained parameter values. During training, the algorithm runs through many iterations, adjusting the weights (W) and biases (b) based on data. Once training is complete, this final equation — with its learned parameters locked in — is the model.

Raw algorithm → many iterations of adjustment → parameters stabilise → Model ✅
🏫

Supervised Learning

You teach with labeled examples — like a student studying with an answer key.

Input → [Model] → Output → Correct label given
Task Types
  • Classification (cat vs. dog)
  • Regression (predict house price)
Algorithms
  • Linear / Logistic Regression
  • Decision Trees, Random Forest
  • SVM, Neural Networks
Real-World Uses
Spam FilterImage RecognitionLoan Prediction
🔍

Unsupervised Learning

Gets no labels at all. Exploratory — the model finds its own structure in raw data.

Input → [Model] → Patterns — No labels provided
Task Types
  • Clustering (group similar items)
  • Dimensionality reduction (compress)
Algorithms
  • K-Means Clustering, DBSCAN
  • PCA, t-SNE
  • Autoencoders
Real-World Uses
Customer SegmentationAnomaly DetectionRecommendations
🏆

Reinforcement Learning

No dataset — an agent takes actions, receives rewards or penalties, and learns the best strategy through experience.

Action → Reward / Penalty → Agent updates strategy
Task Types
  • Policy optimisation
  • Game playing & robotic control
Algorithms
  • Q-Learning, Deep Q-Network (DQN)
  • PPO (Proximal Policy Optimisation)
  • RLHF — used in LLMs!
Real-World Uses
AlphaGoRobotics ControlAd Bidding

🌟 Special: RLHF — How ChatGPT Was Made Human-Friendly

Reinforcement Learning from Human Feedback (RLHF): Human reviewers rate responses → feedback trains a reward model → the LLM is updated to match human expectations — factual, helpful, and safe.

📖 Case Study: How ChatGPT Uses All Three Learning Types

1. Unsupervised Pre-training (GPT-3)

Trained on massive text by predicting the next word — no labels needed. Original GPT-3 had 175B parameters.

2. Supervised Fine-tuning (→ ChatGPT)

Fine-tuned on (prompt → correct response) pairs so the model learns to follow instructions — the leap from raw GPT-3 to ChatGPT.

3. Reinforcement via RLHF

Human reviewers rate responses → reward model trained → LLM updated to match human expectations. Makes answers feel natural and safe.

📋 PM Guide: How to Think About Data

🎯 Define Scope Upfront

Be clear about what your system will and won't do from the start. This saves significant time and cost downstream.

🔒 Privacy & Compliance

Know the law — GDPR (Europe) and CCPA (California). What can you collect? How long can you store it?

🗃️ Data Sourcing

Evaluate trusted vs. non-trusted data sources — they have very different implications for model quality.

💰 Cost Evaluation

Evaluate the cost to build and maintain your data pipeline upfront — data is expensive to label, store, and keep fresh.

What an AI System Needs

💬 The Core Summary

"AI is the ability of machines to learn, reason, perceive, and make decisions in ways that mimic human intelligence. In practice, building a reliable AI system (Agents) requires four key pillars: Knowledge (what it knows), Data (what it learns from), Tools (what it can act with), and Guardrails (how it stays safe and ethical)."

🗂️

Knowledge

Domain understanding the model encodes — via training, RAG (Retrieval-Augmented Generation), or knowledge graphs.

📊

Data

The fuel for training and fine-tuning ML models. Essential for modern AI systems.

🔧

Tools

APIs, search, code execution — they extend what the model can do beyond generating text.

🛡️

Guardrails

Safety, ethics, and compliance rules applied around the model. This is the governance layer — not AI itself.

Inference & Model Assessment

What is Inference?

After training, you make the model available to customers. Inference is the process of sending new input to the trained model and receiving a prediction — the "production" phase of an AI system.

Accuracy
Correct Predictions ÷ Total
How often is the model correct overall?
Recall
TP ÷ (TP + FN)
How many real positives did we catch?
Precision
TP ÷ (TP + FP)
Of predicted positives, how many were actually correct?

⚠️ PM Tip — Set Benchmarks Based on Customer Needs

For example, automating a task requires high precision — you need to be confident the model is correctly identifying outcomes before removing human oversight. Always align your benchmarks to what matters most to your customer.

Prompt Engineering

What is a Prompt?

A prompt is a set of instructions provided to an AI model to get the best output. The quality of your prompt directly determines the quality of the AI's response.

🏆 10 Best Practices

1

Be Specific

Include all relevant details and context in your request

2

Supply Examples

Give the model sample input/output pairs for context

3

Include Relevant Data

Provide the data the model needs to reason with

4

Specify Output Format

e.g., "Respond in JSON" or "Use bullet points"

5

Use Positive Instructions

Tell it what TO do, not just what not to do

 

6

Assign a Persona

"You are an expert product manager…"

7

Chain of Thought

"Think step by step before answering"

8

Break Down Complex Tasks

Split large tasks into smaller, specific sub-tasks

9

Acknowledge Limitations

Tell the model what it might not know

10

Take an Experimental Approach

Iterate and test — prompting is a craft, not a formula

📐 The RITG Prompt Framework

Structure every important prompt using these four components:

R — Role

Define the persona. e.g., "You are a senior data scientist…"

I — Instruction

Set the overall goal. e.g., "Analyse the following customer feedback…"

T — Task

List step-by-step specific activities for the model to perform

G — Guardrails

Set boundaries: preferred tone, output format, and fallback if ambiguous

Choosing the Right AI Model

3-Step Framework

A structured way to select the right model for your GenAI application, aligned to your customer journey and business constraints.

1

List Tasks & Jobs-to-be-Done

Based on the customer journey, identify every specific task your app needs to perform. Be granular — "summarise a report" and "draft a reply email" are different tasks with different requirements.

2

Create Requirements Across Key Dimensions

For each task, define requirements across: Reasoning vs. Understanding, Budget, Latency, Time-to-Market, Tools Needed, and your Team's AI expertise.

3

Look Up Relevant Benchmarks

Find benchmarks that show which model performs best on your specific task. Use artificialanalysis.ai and always review the model's Model Card before committing.

📄 What are Model Cards?

Concise documents introduced by Google that provide key information about an ML model. Always review before selecting a model for production use.

Claude Chat vs Cowork vs Claude Code

💬 The Simple Mental Model

Claude Chat is for thinking with Claude.  ·  Cowork is for delegating to Claude.  ·  Claude Code is for building with Claude.

Claude Chat

💬 Think

You Chat → Claude Responds

General-purpose AI assistant at claude.ai. Handles writing, analysis, coding help, research, and brainstorming in a back-and-forth conversation.

Free tier includes: Memory across conversations, file creation, code execution, web search, Slack & Google Workspace integration
Web & Mobile Conversation Tool
Cowork

🤝 Delegate

Plain Language → Task Done

Built into the macOS Claude desktop app. Give Claude access to a folder and delegate file tasks — expense reports, writing reports from notes, reorganising folders.

Best for: Business analysts, operations managers, non-technical teams handling repetitive file tasks
No Setup Required Isolated VM
Claude Code

⚙️ Build

Describe Problem → Code Delivered

Command-line coding agent for developers. Runs in your terminal or VS Code. Reads your entire codebase for full project context — handles refactoring, debugging, tests, and Git workflows.

Advantages over Chat: More precision, speed, control, extensibility, and reliability for complex tasks
Terminal / VS Code Full Codebase Context

🎯 PM Decision Guide — Which Claude Should You Use?

Use Claude Chat when…
  • You need a quick answer or brainstorm
  • You're drafting content or analysing a document
  • You want a back-and-forth conversation
  • You're on mobile or web with no local setup
Use Cowork when…
  • You want Claude to work on files on your Mac
  • You're not technical and don't want to script
  • The task is file-based and repetitive
  • You need it isolated and safe from the internet
Use Claude Code when…
  • You're a developer working in a codebase
  • You need full project context across files
  • You want Git integration and terminal control
  • The task is complex and requires precision

⚠️ Key Limitation to Know

Claude Chat does not take autonomous action outside the chat window — it's a back-and-forth conversation tool only. For autonomous file tasks or code changes, you need Cowork or Claude Code respectively.

AI Product Manager Approach

🎯 Core AI PM Objectives

a. Translate Business Needs

Turn business problems into clearly defined, solvable AI problems with measurable outcomes.

b. Target Users & Workflows

Identify the specific users and daily workflows where AI-driven products can create the most value.

c. Define Success Metrics

Set clear, actionable success metrics for AI solutions before building — time saved, accuracy, cost reduction.

d. Build Your MOAT

Margin · Operations · Advantage · Team — the four dimensions of durable competitive advantage.

🏅 MOAT Framework

Margin  ·  Operations  ·  Advantage  ·  Team — the four dimensions of building a competitive AI product that is hard to copy.

1

Identify Opportunities

Focus on repetitive daily or weekly tasks that are core to the business. Ensure sufficient data & resources exist, verify regulatory compliance, and confirm the solution can scale with minimal human intervention.

2

Articulate the Problem

Define the customer persona & their challenges. Articulate measurable consequences of the problem. Highlight gaps in current SaaS or rule-based solutions, and explain why generic AI tools (ChatGPT, Copilot) fall short.

3

Identify the Solution

Introduce your solution and what it offers users. Account for AI limitations (hallucinations, reasoning, planning). Plan for human oversight with quality assurance until AI results are reliable. Build MOAT through deep integrations.

4

Define Success Metrics

Focus on time saved, cost reduction, and accuracy improvement. Use the two-tier metrics framework:

🟢 LEVEL 1 — Primary Metrics
  • Error Rate (%)
  • User Feedback
  • Helpfulness
  • Honesty (Accuracy of Facts)
  • Harmlessness
🔵 LEVEL 2 — Secondary Metrics
  • Cloud / AI Infrastructure Cost
  • Avg Tokens Consumed Per Job
  • Latency — P95 / P99 Response Time
  • Throughput — TPM / QPS / TPS
  • Recovery Rate & Resilience

What is CLAUDE.md?

Definition

CLAUDE.md is a markdown file you add to your project root that Claude Code reads at the start of every session. It is specific to Claude Code (the terminal/IDE tool) — not Claude Chat. Your CLAUDE.md file becomes part of Claude's system prompt, so every conversation starts with your project context already loaded.

📁

Project-Level File

Lives in the project root directory. Contains rules, design systems, commands, and gotchas specific to that codebase. Loaded automatically every session.

🌐

Global File

Lives at ~/.claude/CLAUDE.md. Contains personal preferences that hold across all your projects — both can coexist and are both loaded.

📝 What Goes in CLAUDE.md?

🏗️ Project Context
  • Project name, owner, purpose
  • Tech stack and architecture overview
  • File structure and naming conventions
  • Navigation flow between pages
🎨 Design Standards
  • CSS token system and colour palette
  • Typography rules and font imports
  • Component patterns (cards, callouts, navs)
  • Animation and interaction rules
⚙️ Commands & Workflows
  • How to run tests, builds, and linting
  • How to start the local server
  • Deploy and release procedures
  • Git workflow conventions
⚠️ Gotchas & Rules
  • What NOT to do (forbidden patterns)
  • Files that must never be renamed/deleted
  • Project-specific warnings and edge cases
  • Future pages and planned extensions

💡 Key Benefit

Eliminates the need to re-explain your project at the start of every session. Claude reads CLAUDE.md automatically, so it always knows your standards, constraints, and conventions before writing a single line of code.

What is SKILLS.md & How It Differs from CLAUDE.md

Definition

Skills are reusable workflows you invoke with /skill-name, stored under .claude/skills/ at the project level or ~/.claude/skills/ for global availability. They are on-demand, task-specific instruction sets — you invoke them when you need them, rather than them loading automatically every session.

🔑 CLAUDE.md vs SKILLS.md — Key Differences

Dimension 📋 CLAUDE.md ⚡ SKILLS.md
LoadedAutomatically — every sessionOn demand — when invoked
PurposePersistent project contextReusable task workflows
ScopeProject identity and rulesSpecific, repeatable tasks
InvocationAlways active/skill-name command
Best ForHow this project worksHow to do X task
ExampleDesign system, file naming rules, colour tokens/create-project, /create-week-summary

📋 Think of CLAUDE.md as…

The project's constitution — the always-on rules and context that define what the project is, how it looks, and what is and isn't allowed. Claude reads it before doing anything.

Always Active Project Identity Standards Gate

⚡ Think of SKILLS.md as…

The project's playbook — a library of on-demand recipes you call when you need to do a specific task. Each skill is a locked, consistent pipeline that uses CLAUDE.md as its quality gate.

On Demand Task Pipeline Repeatable

🏅 PM Tip — When to Create a Skill

Create a skill when you find yourself giving Claude the same multi-step instructions more than twice. If you're saying "create a new project page, add it to the hub, update the nav, and verify the design tokens" repeatedly — that's a skill. Codify it once in SKILLS.md and invoke it forever after with a single command.
